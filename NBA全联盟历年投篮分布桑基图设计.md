# NBA全联盟历年投篮分布桑基图设计.md
## 1. 整体定位与页面目标
### 产品定位
独立宏观趋势分析页面，采用桑基图可视化全联盟多年投篮出手结构流转规律，覆盖23个完整赛季数据，量化不同出手区域资源分配长期变迁，直观呈现联盟战术风格迭代过程。
### 核心分析问题
1. 历年联盟出手总量在篮下、中距离、三分核心区域的流量占比逐年流转变化；
2. 可视化篮球时代演变：90年代中距离主导阶段、小球时代三分爆发阶段、现代空间内线转型阶段；
3. 支持多维度筛选、自定义时间区间对比、时序动态播放，区分总出手、命中数两套数据展示口径；
4. 统一底层数据标准，提供联盟、球队、明星球员三层数据粒度自由切换。
### 页面核心价值
桑基图擅长展示分类流量分配、结构迁移、权重占比演变，可量化各出手区域出手权重变化，清晰还原二十余年联盟整体投篮策略迭代趋势。

## 6. 数据分区与流量计算规则

### 区域划分标准（基于原始投篮坐标）

原始投篮原点为进攻篮筐正下方，坐标范围 X：-250~250，Y：0~470

1. 篮下区域：Y ≤ 80 近筐出手
2. 中距离区域：80 ＜ Y ≤ 230 两分区域
3. 三分区域：Y ＞ 230 外线三分区域
五分区精细拆分：
4. 近筐：Y ≤ 80
5. 短两分：80 ＜ Y ≤ 150
6. 长两分：150 ＜ Y ≤ 230
7. 常规三分：230 ＜ Y ≤ 380
8. 超远三分：Y ＞ 380

### 流量聚合逻辑

1. 按赛季、筛选主体（联盟 / 球队 / 球员）、出手区域三层分组聚合；
2. 每组聚合产出：总出手 count、命中数 make、区域命中率 fg\_pct=make/count；
3. 桑基连线宽度映射 count 数值，节点边框深浅映射 fg\_pct；
4. 前端仅做视觉映射，原始聚合数据由后端预处理输出，不修改源数据结构。

## 8. 数据预处理参数规范

表格

| 预处理参数 | 参数值 | 说明 |
| --- | --- | --- |
| 最小过滤阈值 - 联盟 / 球队 | count ≥ 5 | 过滤极低频次分区数据，避免无效细流量 |
| 最小过滤阈值 - 单球员 | count ≥ 3 | 单球员样本量偏小，降低过滤门槛 |
| 赛季总数量 | 23 个 | 覆盖 1997-98 至最新赛季 |
| 单文件预估体积 | ~400KB | 包含联盟、30 支球队、12 位球员分区聚合数据 |
| 全部文件总占用 | ≤10MB | 控制前端静态资源加载体积 |
| 赛季转换逻辑 | 比赛日期转赛季 | 10 月及以后归当年新赛季，1-9 月归上一赛季 |

### 赛季转换 Python 逻辑

```
# 根据比赛日期数字(YYYYMMDD)生成赛季标签
game_date = 19980102
year = game_date // 10000
month = (game_date // 100) % 100
if month >= 10:
    season = f"{year}-{str(year+1)[2:]}"
else:
    season = f"{year-1}-{str(year)[2:]}"
```

## 9. 前端数据加载与渲染流程

1. 用户选择起止赛季、筛选主体（联盟 / 球队 / 球员）、分区模式；
2. 前端状态管理器校验本地缓存，未缓存则发起 fetch 请求加载对应赛季`sankey_season_{season}.json`；
3. 缓存写入全局状态池，重复切换赛季无需重复请求；
4. 根据选中主体类型提取对应 areas 分区数组，按时序合并多赛季数据；
5. 基于 count 数值计算桑基连线宽度，fg\_pct 映射节点边框深浅；
6. SVG 桑基组件监听数据变更，自动重绘完整时序流量图；
7. 开启播放动画时，逐赛季切换时序节点，同步更新流量连线宽度与颜色。

## 10. 上线前数据核验清单

预处理脚本执行完成后逐项校验：

- 23 份赛季 JSON 文件全部生成，无空文件、损坏文件
- 每份文件内 league.areas 数组不为空，存在有效分区数据
- 所有 fg\_pct 数值严格落在 0~1 区间，保留 3 位小数
- 各分区 count 数值大于对应最小过滤阈值
- 12 位预设球星在有出手样本的赛季中均存在对应数据节点
- 全部数据文件总存储体积不超过 15MB
- 本地环境单文件接口请求耗时＜200ms，加载无卡顿
- 多赛季时序拼接无坐标越界、流量连线错位问题
- 分区切换（三分区 / 五分区）聚合逻辑无数据丢失、重复计算
- 播放动画时序切换流畅，数值、颜色同步更新无渲染延迟
