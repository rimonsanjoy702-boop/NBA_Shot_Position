## 2. 核心交互规则

### 2.1 两层响应，严格区分

| 交互方式 | 触发效果 |
|---------|---------|
| **Hover** 任意可交互元素 | **仅显示 tooltip**，图表视觉完全不变 |
| **Click** 节点/bin/格子/折线 | 被选中元素 + 其上下游关联元素保持原视觉；该图表其余部分 dim（饱和度降至 `saturate(0.08)` + 透明度降至 25-35%） |
| **Click** 已选中元素 | 取消选中，全部恢复全貌 |
| **Click** 图表空白区域 | 取消该图表所有选中 |
| **切换选择器**（赛季/粒度/实体） | 清空所有选中态，恢复默认全貌 |

### 2.2 Dim 视觉规范

| 维度 | 选中/关联元素 | 其余元素（dim） |
|------|-------------|----------------|
| 颜色饱和度 | 保持原色 | `saturate(0.08)` 接近灰度 |
| 不透明度 | 保持原值 | 降至原值的 25-35% |
| 边框/描边 | 保持原值 | 不变（已被饱和度降低覆盖） |
| 额外效果 | 无（不放大、不加亮、不闪烁、不加虚线标记） | — |

SVG 实现用 `<feColorMatrix type="saturate" values="0.08"/>` + `<feFuncA type="linear" slope="0.35"/>`；ECharts 用 `series` 数据动态替换；D3 用 `transition` + CSS `filter`。

## 3. 左右数据槽模型（黄金三角共享）

Hexbin、桑基图、时间-FG%衰减曲线三者各维护两组独立数据视图——**左槽(Left Slot)** 和 **右槽(Right Slot)**。每个槽是一个完整的数据上下文：

```
Slot = { season: string, scope: 'league'|'team'|'player', entityId?: number, entityLabel?: string }
```

- **Hexbin**：两个槽同时在球场 SVG 上渲染（左半场/右半场），各自有独立的选择器
- **桑基图**：同一时刻仅展示一个槽的数据，通过左/右 Tab 切换
- **衰减曲线**：两个槽的数据同时渲染（左槽→实线组，右槽→虚线组），**自身没有选择器**，完全跟随 Hexbin 对应槽的选择

### 3.1 主动侧（Active Side）机制

衰减曲线和桑基图 L1 的时间 bin 点击效应，作用于**当前主动侧**（最后被用户操作过的选择器所在侧）。默认主动侧为**左侧**。

| # | 触发 | 效果 |
|---|------|------|
| F1 | 用户操作 Hexbin 左侧选择器 | `activeSide = 'left'` |
| F2 | 用户操作 Hexbin 右侧选择器 | `activeSide = 'right'` |
| F3 | 用户切换桑基图到左 Tab | `activeSide = 'left'` |
| F4 | 用户切换桑基图到右 Tab | `activeSide = 'right'` |
| F5 | 默认（页面加载） | `activeSide = 'left'` |

### 4.4 衰减曲线两侧视觉分配规则

衰减曲线始终同时渲染两条折线（左槽实线 + 右槽虚线），但视觉权重由主动侧决定：

| 主动侧 | 左侧曲线组（实线） | 右侧曲线组（虚线） |
|--------|-----------------|-----------------|
| left | 保持原色、原线宽 | dim（灰度化 + opacity 0.35） |
| right | dim（灰度化 + opacity 0.35） | 保持原色、原线宽 |

数据层面：两侧曲线各自跟随 Hexbin 对应侧的选择器数据，**互不影响**（H2/H4 中只更新被操作侧的数据）。

## 6. 桑基图节点点击的正确 Dim 语义

### 6.1 "自身 + 上下游关联" 的定义

点击桑基图某个节点时，"保持原色"的元素集合为：

```
clicked node
  + 所有以 clicked node 为 source 的下游连线 (outgoing links)
  + 所有以 clicked node 为 target 的上游连线 (incoming links)
  + 所有下游连线指向的 target 节点 (递归)
  + 所有上游连线的 source 节点 (递归)
```

即：**被点击节点所在的完整连通子图**保持原视觉，其余全部 dim。

具体到各层：

| 点击层 | 保持原色 |
|--------|---------|
| L1 | 该 L1 节点 + 其所有 L1→L2 连线 + 被连到的 L2 节点 + 后续 L2→L3→L4 全部连线及节点 |
| L2 | 该 L2 节点 + 指向它的 L1→L2 连线及 L1 节点 + 它发出的 L2→L3→L4 全部连线及节点 |
| L3 | 该 L3 节点 + 指向它的 L2→L3 连线及 L2 节点 + 其上游 L1→L2 连线及 L1 节点 + 它发出的 L3→L4 连线及 L4 节点 |
| L4 | 该 L4 节点 + 指向它的 L3→L4 连线及 L3 节点 + 其上游 L2→L3 连线及 L2 节点 + L1→L2 连线及 L1 节点 |

### 6.2 当前实现的错误

当前 `SankeyChart.vue` 中 `focusedNodeId` 模式：

- 连线 dim 时仅检查 `link.source.id === focusedNodeId || link.target.id === focusedNodeId`（仅一层邻接），**没有递归追踪上下游完整路径**
- 节点 dim 时仅检查 `focusedNodeId !== node.id`（只有被点击节点本身保持），**上下游关联节点没有豁免 dim**
- `linkOpacity` computed 中多个 `if` 块（`focusedNodeId`、`selectedTimeBin`、`selectedZone`）**未统一逻辑**

**需要修复为**：点击节点 → 计算完整上游+下游连通集合 → 集合内元素保持原色，其余全部 dim。

## 9. 与现有实现的主要差异

| 方面 | 现有代码 | 设计方案要求 |
|------|---------|-------------|
| 桑基图左右槽 | `DashboardPage` 本地 state | 提升到 Pinia Store (`leftSlot`/`rightSlot`)，Hexbin/衰减曲线可读写 |
