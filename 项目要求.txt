1. 项目基本信息与分工
	分工与人数：属于小组作业（Teamwork），每组固定为 3名成员。 
	项目分值：占课程总成绩的 50%，另外有 20% 为项目展示/答辩分数。 
	选题要求：自由选题，但数据集与主题需符合核心价值观。选好数据后，必须与分配到的带组助教进行确认，也可以向助教寻求帮助。 
	人员职责要求：团队内部务必明确分工，每个人都必须参与编程、数据清洗或使用可视化工具等实质性工作，绝对不允许出现某个人“只负责答辩”的情况。 
2. 项目形式与技术要求（二选一）
你可以根据团队的技术背景，在以下两种形式中任选一种完成。无论选择哪种，都必须包含数据预处理、可视化、数据洞察解读（Insight）这三个核心步骤： 
	形式 A：使用可视分析工具进行数据分析
	选用工具：Python, Tableau 等。 
	核心要求：要求能够传达深刻的数据内涵，讲述完整的数据故事，并在展示中体现出尽可能深刻的“数据洞察（Insight）”。 
	形式 B：自己设计并开发可视分析系统（有5分Bonus）
	核心要求：如果自己开发系统，可以获得 5分的附加分（Bonus）。系统必须支持多视图（数量大于等于2）的可视化呈现和探索，具备跨视图的交互（联动）能力，并能够展示出完整的应用案例。 
💡 什么是课程要求的“数据洞察解读（Insight）”？ 课程明确指出：Insight = 可视化 + 推理。不能只是一味地堆砌和罗列数据，而是要通过可视化视图，解释“在哪里发现了什么规律”以及“为什么会出现这个规律”，分析出隐藏在趋势背后的深层原因和影响因素。 
3. 作业提交物要求
作业由小组以单位提交一次即可，包含以下三项内容： 
	答辩 PPT。 
	系统或数据分析的代码包：代码打包时须包含一个运行说明文档 README.md。注意：如果使用 Node.js 开发，绝对不要把 node_modules 文件夹打包进去！ 
	简短的报告：必须控制在两页 A4 纸以内（支持 PDF、Word 或 TXT 格式，无严格格式要求），报告中必须附上商量好的人员贡献表（包含姓名、职责描述、贡献度百分比，平分写33%即可）。 
4. 线下展示与答辩要求
	展示时间：7月24日 13:50 - 15:10。 
	PPT提交截止：7月23日 23:59 之前。 
	展示规则：
	限时 5分钟/组，展示后包含问答（Q&A）环节。 
	小组全体成员必须到场参与答辩，全体同学均需到场聆听（出勤占10%）。 
	展示内容：须阐述数据背景、数据预处理、可视化、小组分工以及数据洞察（Insight）。如果开发了可视分析系统，必须在现场或在 PPT 中播放录制好的系统运行 Demo，展示系统的完整功能和得出的分析结论。 
5. 自建前端系统技术规范（仅针对选择“自建系统”的小组）
如果小组选择搭建前端界面，须遵守以下工程规范：
	环境要求：Node.js 版本必须 ≥18.0，官方推荐统一使用 v22.17.0 (LTS)。 
	官方脚手架：课程在 GitHub 上提供了一个基于 Vue 框架、使用 pnpm 包管理工具的官方模板（仓库：cyber-loafing/2025 ZJU-SummerSchool-project-template），里面包含了一套视图联动的解决方案，供开发参考。 
	推荐工具链：
	编辑器：VS Code 或 WebStorm 
	数据/组件库：ECharts, Vue-ECharts, ElementPlus, Antd 
	复杂可视化：原生 SVG, D3.js, Konva, PixiJS 
	代码协作与原型：Git, Figma
