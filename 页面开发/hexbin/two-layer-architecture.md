# Hexbin 全场对比热力图 · 两层分离架构方案

> **写给前端开发队友的球场底图绘制指南**
>
> 版本：v1.0 | 2026-07-21

---

## 核心思路

Hexbin 热力图页面采用**两层 SVG 叠加**架构：

```
┌──────────────────────────────────────────────────────────┐
│  控制栏                                                   │
│  [球员/球队 v] [赛季 v]    [固定标尺] [逐年播放]    [球员/球队 v] [赛季 v] │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ┌────────────────── 叠加区域（CSS 定位）─────────────┐   │
│  │                                                    │   │
│  │   Layer 0: 球场底图层（静态 SVG，只画一次）          │   │
│  │   ├── 木地板底色 (#c88d3c 系暖木色)                  │   │
│  │   ├── 边线 / 中线                                   │   │
│  │   ├── 三分线 / 禁区                                  │   │
│  │   ├── 跳球圈                                        │   │
│  │   └── 篮筐图标 ◯                                    │   │
│  │                                                    │   │
│  │   Layer 1-left: 左半场 Hexbin（数据变化时重绘）       │   │
│  │   └── 六边形网格，数据来源于 hexbin JSON             │   │
│  │                                                    │   │
│  │   Layer 1-right: 右半场 Hexbin（数据变化时重绘）     │   │
│  │   └── 六边形网格，数据来源于 hexbin JSON             │   │
│  │                                                    │   │
│  └────────────────────────────────────────────────────┘   │
│                                                          │
│  图例: 投篮密度 [○ ──大小── ○]   命中率 [浅 ──深浅── 深]    │
└──────────────────────────────────────────────────────────┘
```

### 为什么两层分离

| 好处 | 说明 |
|---|---|
| **球场只画一次** | 不管用户换赛季/球员，球场线永不重绘，零闪烁 |
| **性能** | 数据切换只更新 Hexbin 层，渲染量减半 |
| **分工并行** | 球场底图一个人画，Hexbin 另一个人画，不耦合 |
| **维护性** | 改球场线不影响数据逻辑，反之亦然 |

### SVG viewBox 约定

球场全局 viewBox: `"0 0 1880 500"`，**响应式**（根据容器宽度动态缩放，`preserveAspectRatio` 控制）。

## 球场底图规格（队友做的部分）

### 坐标系

使用 viewBox 坐标系（不是原始数据坐标系）。

```
viewBox="0 0 1880 500"

左篮筐中心:  (150, 250)    ← 基准点
右篮筐中心:  (1730, 250)   ← 对称
中场线 X:    940
```

### 球场关键线（按 viewBox 坐标）

以下坐标换算基于 **NBA 标准球场 94ft×50ft，viewBox 宽度 1880 对应全场宽度**（比例尺 ≈ 20 viewBox-unit/ft）：

| 元素 | viewBox 坐标 | 说明 |
|---|---|---|
| **边线（全场）** | (20, 20) → (1860, 480) | 矩形框 |
| **中线** | X=940，Y 从 20 到 480 | 竖向分割左右半场 |
| **左篮筐中心** | (150, 250) | 篮筐图标 ◯，半径 8px |
| **右篮筐中心** | (1730, 250) | 篮筐图标 ◯，半径 8px |
| **左禁区** | 矩形: X=150±80, Y=250±90 | 罚球区 |
| **右禁区** | 矩形: X=1730±80, Y=250±90 | 罚球区 |
| **左三分线** | 圆心(150,250)，弧半径 476（≈23.75ft），截于底线 | 只画弧线部分 |
| **右三分线** | 圆心(1730,250)，弧半径 476 | 对称 |
| **中场跳球圈** | 圆心(940,250)，半径 120（≈6ft） | 两个半圆 + 中心点 |
| **左半场跳球圈** | 圆心(150,250)，半径 120 | 罚球线顶端半圆 |
| **右半场跳球圈** | 圆心(1730,250)，半径 120 | 同上 |

> ⚠️ 以上坐标为估算值。建议队友用 D3.js 或手写 SVG `<path>` 绘制，参考 [NBA 官方球场尺寸](https://cdn.nba.com/manage/2024-10/2024-25-Official-NBA-Rule-Book.pdf) 精确微调。

### 视觉参数（引用自共享视觉规范）

```css
木地板底色:  #b87c3a (暖木棕) 或保留 rgba 透明让页面深色背景透出
球场线颜色:  rgba(255,255,255,0.3)  ← 半透白，不抢数据注意力
线条宽度:    1.5px (边界), 1px (内部线)
篮筐颜色:    #ff6b6b (语义色-rim 红)
```

### 输出物

一个 `<svg>` 标签，`id="court-base"`，独立写在组件中（如 `CourtBase.vue`）。

## Hexbin 数据层（我们做的部分）

### 数据来源

`public/data/hexbin_season_{season}.json` — 23 个赛季文件。

每个 hex cell: `{ x, y, count, fg_pct }`

### 坐标变换（JSON 坐标 → viewBox 坐标）

**JSON 数据坐标保持不变**，仅在前端渲染时做 swap。

左半场（篮筐在左，hex 向右展开）：
```javascript
viewX = 150 + y_original;   // Y → 水平向右展开
viewY = 250 + x_original;   // X → 垂直方向
```

右半场（篮筐在右，hex 向左展开）：
```javascript
viewX = 1730 - y_original;  // Y → 水平向左展开（取负）
viewY = 250 + x_original;   // X → 垂直方向（不变）
```

### 视觉编码（引用自共享视觉规范）

| 维度 | 编码 | 参数 |
|---|---|---|
| 投篮量 | 六边形外接圆半径 | 动态标尺：min 4px ~ max 22px，根据 (count, fg_pct) 映射 |
| 命中率 | 颜色 | 红→橙→黄→绿→翠绿色阶 |
| FG%=0~30% | `#ff6b6b` | 非常低效 |
| FG%=30~40% | `#f4a460` | 低效 |
| FG%=40~50% | `#f4d03f` | 中等 |
| FG%=50~60% | `#5cdb8b` | 高效 |
| FG%=60~100% | `#00d2a0` | 非常高效 |

颜色标尺：两半场默认共用动态标尺（基于左右数据并集），可选固定 0~100%。

### 交互

| 交互 | 行为 |
|---|---|
| 悬停 hex | tooltip 显示：区域名、FG%、出手次数、占该区域占比 |
| 点击 hex | 无（Hexbin 不支持下钻到单个 hex） |

## 技术栈

| 层 | 技术 | 组件名（建议） |
|---|---|---|
| 球场底图 | 纯 SVG 手写（或 D3 one-shot） | `CourtBase.vue` |
| 左半场 Hexbin | D3.js + Vue | `HexbinHalf.vue`（props: side="left"） |
| 右半场 Hexbin | D3.js + Vue | 同上组件，props: side="right" |
| 控制栏 | Element Plus | `HexbinControls.vue` |
| 图例 | 纯 HTML/CSS | `HexbinLegend.vue` |
| 页面 | Vue Router | `HexbinPage.vue` |

## 给队友的球场底图 checklist

- [ ] SVG viewBox="0 0 1880 500"，`preserveAspectRatio="xMidYMid meet"`
- [ ] 木地板底色（建议 `#b87c3a` 或略深，适配深色页面背景）
- [ ] 边线矩形 (20, 20) → (1860, 480)
- [ ] 中线 X=940
- [ ] 左右篮筐图标 ◯
- [ ] 左右三分线弧线
- [ ] 左右禁区矩形
- [ ] 中场跳球圈
- [ ] 所有线用 `rgba(255,255,255,0.3)`，不抢焦点
- [ ] 不使用任何硬编码色值，引用 `var(--xxx)` CSS 变量
- [ ] 独立组件：接受零 props，永不重绘
