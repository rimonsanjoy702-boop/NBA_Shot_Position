### 为什么是单页面全套而非多 Tab 拆分

| 单页面全套 | 多 Tab 拆分 |
|---|---|
| 曲线 + 卡片 + 柱状图同时可见，形成"看 → 读 → 验"闭环 | 三者在不同页面，需要记忆和切换 |
| 滑块拖动时三区同步更新，视觉反馈统一 | 需手动同步选中状态，容易断联 |
| 答辩时只需操作滑块，所有图自动跟变 | 需要切页面打断叙事节奏 |
| 5分钟演示容纳完整故事线 | 切页浪费宝贵的演示时间 |

### 三层视图的分工

```
第一层：KDE 曲线 — "看到什么"
   → 直观展示距离分布从单峰到双峰的形态变化
   → 白色当前赛季曲线在橙(1998)和蓝(2020)之间滑动

第二层：统计卡片 — "读到什么"
   → 把视觉感受量化为精确数字
   → 每张卡片标注 vs 1998 的变化量（Δ）

第三层：双峰指数柱状图 — "验证什么"
   → 证明这不是个别赛季的波动，而是持续24年的单向趋势
   → 橙色高亮柱标记当前赛季在全时段中的位置
```

## 2. 视觉编码

| 维度 | 编码方式 | 说明 |
|---|---|---|
| 赛季演进 | 曲线颜色（橙→白→蓝） | 1998 暖橙 = 早期，2020 亮蓝 = 近期，当前赛季纯白高亮 |
| 概率密度 | Y 轴高度 + 半透明面积填充 | 峰越高 = 该距离的出手越密集 |
| 投篮距离 | X 轴位置 | 0ft（篮下）到 35ft（超远三分） |
| 双峰程度 | 柱状图高度 + 颜色深度 | 灰色=低双峰（早期），亮蓝=高双峰（近期），橙色=当前赛季 |
| 区域语义 | 背景色块 | 篮下=暖红半透明，中距离=灰色半透明，三分=蓝色半透明 |
| 数据变化 | 卡片 Δ 值 + 颜色 | 正向变化绿色 ↑，负向变化红色 ↓ |

> 一个页面同时回答三个问题：**"分布怎么变"**（曲线形态）、**"变了多少"**（卡片数字）、**"是趋势还是偶然"**（柱状图轨迹）。

## 3. 数据粒度

### 三级聚合

| 层级 | 数据量 | 聚合方式 |
|---|---|---|
| 原始投篮 | 4,729,512 行 | 每行 = 一次投篮的 Shot Distance (0-89 ft) |
| 赛季级 KDE | 23 赛季 × 161 评估点 | Gaussian KDE, bandwidth=0.6, step=0.25ft |
| 赛季级指标 | 23 赛季 × 7 字段 | 双峰指数、篮下/中距离/三分占比、出手数、峰/谷值 |

### KDE 评估点参数

| 参数 | 值 | 说明 |
|---|---|---|
| 距离范围 | 0 ~ 40 ft | 覆盖全部有效投篮（40ft 以上忽略） |
| 评估步长 | 0.25 ft | 每 0.25ft 一个采样点，共 161 点 |
| 带宽 (bandwidth) | 0.6 | 偏小带宽使曲线保留更细的结构特征 |
| 归一化 | 总面积 = 1.0 | 确保不同赛季的曲线可比（出手总数不同） |

### 双峰指数公式

```
bimodality_score = (rim_peak + three_peak) / (2 × mid_valley)

rim_peak   = KDE 在 0-3ft 区间的最大密度值
three_peak = KDE 在 22-25ft 区间的最大密度值
mid_valley = KDE 在 10-18ft 区间的最小密度值
```

| 双峰指数 | 含义 | 典型赛季 |
|---|---|---|
| < 6.0 | 近似单峰分布，中距离仍占主导 | 1997-98 ~ 2001-02 |
| 6.0 ~ 7.0 | 过渡期，两端开始隆起 | 2002-03 ~ 2011-12 |
| > 7.0 | 明显双峰，中距离塌陷 | 2016-17 ~ 2019-20 |

## 5. KDE 计算原理

### 核心概念

KDE（Kernel Density Estimation，核密度估计）是一种非参数密度估计方法。对于一维数据（投篮距离），它通过在每个数据点处放置一个平滑的"核函数"（通常是高斯钟形曲线），然后叠加所有核函数，得到一条平滑的概率密度曲线。

### 数学定义

给定 n 个投篮距离观测值 x₁, x₂, ..., xₙ，KDE 定义为：

```
           1     n
f̂(x) =  ─────  · Σ  K( (x - xᵢ) / h )
         n·h    i=1

其中:
  K(u) = (1/√(2π)) · exp(-u²/2)    — 高斯核函数
  h    = bandwidth                  — 带宽参数，控制曲线平滑度
  n    = 该赛季总投篮次数
```

### 带宽选择

| 带宽 | 效果 | 取舍 |
|---|---|---|
| h=0.3 | 曲线尖锐，保留细节但可能有噪声 | 过拟合风险 |
| **h=0.6** | **适中，保留双峰结构的同时足够平滑** | **选这个** |
| h=1.0 | 曲线非常平滑，但双峰特征可能被抹平 | 欠拟合风险 |

### 为什么用 KDE 而不是简单直方图

| KDE | 直方图 |
|---|---|
| 连续曲线，无 bin 边界突变 | bin 宽度和偏移影响形态 |
| 视觉上"丝滑"，适合动画过渡 | 柱状跳跃，动画感差 |
| 密度值可直接用于双峰指数计算 | 需要额外插值 |
| 答辩视觉冲击力强 | 看起来像"统计作业" |

### 计算流程

```
┌─────────────────┐
│ 原始 CSV (857MB) │
│ 4,729,512 行     │
└────────┬────────┘
         │ chunksize=200000 逐块读取
         ▼
┌─────────────────┐
│ 按赛季分组       │  ← 从 Game Date 推导赛季标签
│ 过滤常规赛       │  ← Season Type = 'Regular Season'
│ 收集距离值       │  ← Shot Distance ∈ [0, 40]
└────────┬────────┘
         │ 每组: numpy.array([d1, d2, ...])
         ▼
┌─────────────────┐
│ scipy gaussian_kde│
│ bw_method = 0.6  │
│ evaluate(0~40ft) │  ← 161 个评估点
│ 归一化总面积=1    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 计算双峰指数     │  ← rim_peak, three_peak, mid_valley
│ 计算区域占比     │  ← rim%, mid%, 3PT%
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ distance_kde.json│  ← 308 KB, 23 赛季
└─────────────────┘
```

### 字段规范

| 字段 | 类型 | 约束 | 说明 |

| `n_shots` | integer | ≥ 500 | 该赛季常规赛总投篮数 |

| `midRangePct` | number | 0.0 ~ 100.0，保留 1 位小数 | 中距离 10-18ft 出手占比 (%) |

| `threePeak` | number | 保留 6 位小数 | 三分区域 KDE 密度最大值 |
| `midValley` | number | 保留 6 位小数 | 中距离区域 KDE 密度最小值 |
| `curve[].distance` | number | 0.0 ~ 40.0，步长 0.25 | 评估点距离 (ft) |
| `curve[].density` | number | 保留 6 位小数 | 该评估点的概率密度值 |

## 7. 预处理参数

| 参数 | 值 | 说明 |
|---|---|---|
| 数据源 | 原始 CSV (857 MB) | Kaggle NBA Shot Locations 1997-2020 |
| 读取方式 | `pd.read_csv(chunksize=200000)` | 逐块读取，每块 20 万行 |
| 赛型过滤 | `Season Type == 'Regular Season'` | 仅常规赛（季后赛样本太少） |
| 距离过滤 | `0 ≤ Shot Distance ≤ 40` | 排除超远后场投篮 |
| 赛季推导 | 从 Game Date (YYYYMMDD) 提取 | 月份 ≥ 10 → 当年-下一年 |
| KDE 算法 | `scipy.stats.gaussian_kde` | 高斯核密度估计 |
| 带宽 | 0.6（自适应） | `bw_method = 0.6 / np.std(data)` |
| 评估点数 | 161 | 0 到 40ft，步长 0.25ft |
| 归一化 | `density / density.sum()` | 总面积 = 1.0 |
| 最小投篮数 | ≥ 500 | 过滤数据不足的赛季 |
| 输出文件 | `distance_kde.json` | 单文件，约 308 KB |
| 输出编码 | UTF-8, `ensure_ascii=False` | 中文注释不转义 |

### 赛季推导逻辑

```python
# 从 Game Date (YYYYMMDD int) 推导赛季标签
year = game_date // 10000          # 如 19980102 → 1998
month = (game_date // 100) % 100   # → 01
if month >= 10:
    season = f"{year}-{str(year+1)[2:]}"     # "1997-98"
else:
    season = f"{year-1}-{str(year)[2:]}"     # "1997-98"
```

### 双峰指数计算

```python
import numpy as np

eval_points = np.arange(0, 40.25, 0.25)  # 161 个评估点
density = kde.evaluate(eval_points)

# 按区间提取密度
rim_mask   = (eval_points >= 0)  & (eval_points <= 3)
mid_mask   = (eval_points >= 10) & (eval_points <= 18)
three_mask = (eval_points >= 22) & (eval_points <= 25)

rim_peak   = float(density[rim_mask].max())
three_peak = float(density[three_mask].max())
mid_valley = float(density[mid_mask].min())

bimodality_score = (rim_peak + three_peak) / (2 * mid_valley + 1e-10)
```

## 8. 前端消费方式

### 数据加载

```
1. KdePage.vue mounted → loadKdeData()
2. data.ts → fetch('demo-data/distance_kde.json')
3. 返回 KdeDataFile 对象 → 存入 ref()
4. 默认选中最后一个赛季 (currentIndex = seasons.length - 1)
5. 显示 2019-20 赛季 → 用户可开始播放或拖动滑块
```

与 Hexbin 模块不同，KDE 数据仅 308 KB，**一次性全量加载**，不需要 Pinia Store 缓存策略。加载后所有 23 赛季数据驻留内存，滑块切换为纯计算属性，响应零延迟。

### 曲线渲染

三条曲线共用同一个 ECharts `line` 系列：

| 曲线 | 数据源 | 颜色 | 透明度 | 线宽 | 说明 |
|---|---|---|---|---|---|
| 1997-98（参考线） | `kdeData[0].curve` | 暖橙 `#ff6b6b` | 0.6 | 2px | 固定不变 |
| 2019-20（参考线） | `kdeData[last].curve` | 亮蓝 `#3498db` | 0.6 | 2px | 固定不变 |
| 当前赛季（主线） | `kdeData[currentIndex].curve` | 白色 `#ffffff` | 1.0 | 3.5px | 随滑块/播放切换 |

两条参考线设置 `animation: false`，当前赛季曲线设置 `animation: true, animationDuration: 400ms`。这样滑块切换时只有白色主线平滑过渡，橙蓝参考线保持静止，形成视觉锚点。

### 区域标注

使用 ECharts `markArea` 在图表背景绘制三个半透明色块：

```javascript
// 篮下区域 (0-3ft)
markArea: {
  data: [[
    { xAxis: '0',  itemStyle: { color: 'rgba(255,107,107,0.08)' } },
    { xAxis: '3' },
  ]],
  label: { formatter: '🏀 篮下 Rim', color: '#ff6b6b' }
}
// 中距离 (10-18ft) — 灰色块
// 三分线 (22-25ft) — 蓝色块
```

### 动画控制器

```
isPlaying (ref<boolean>)
  └─ true → setInterval(500ms/playSpeed) → currentIndex++
     false → clearInterval()

playSpeed (ref<number>)  // 0.5 | 1 | 2
  └─ 切换时: 若 isPlaying → pause() + play() 重建定时器

currentIndex (ref<number>)  // 0 ~ 22
  └─ 绑定到: <input type="range"> 的 v-model
  └─ watch: 驱动 kdeOption 和 bimodalityOption 的 computed 重算
```

### 键盘快捷键

```javascript
ArrowLeft  → currentIndex = max(0, currentIndex - 1)
ArrowRight → currentIndex = min(22, currentIndex + 1)
Space      → e.preventDefault(); togglePlay()
```
