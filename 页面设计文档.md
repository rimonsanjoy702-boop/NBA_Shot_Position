## 1. 概述

### 1.1 页面体系一览

```
/  (HomePage)          首页 · 品牌入口
/dashboard  (DashboardPage)  仪表盘 · 主分析界面（集成 5 类图表）
```

两个页面通过 Vue Router (`src/router.ts`) 注册，使用 `createWebHistory` 模式。

### 1.2 技术栈

| 层面 | 技术 |
|------|------|
| 框架 | Vue 3.5 (Composition API + `<script setup>`) |
| 路由 | Vue Router 4 |
| 状态管理 | Pinia 2 |
| 图表 | ECharts 5 + vue-echarts 7（KDE、三分、时间曲线）<br>自研 SVG 布局引擎（Hexbin、Sankey） |
| UI 库 | Element Plus 2（深色主题覆盖） |
| 构建 | Vite 6 + TypeScript 5.8 |

### 1.3 数据源

所有页面图表共用 23 个赛季数据（1997-98 ~ 2019-20），数据文件在 `public/data/` 下：

| 数据集 | 路径 | 文件数 | 总大小 |
|--------|------|--------|--------|
| Hexbin 热力图 | `public/data/hexbin_season_*.json` | 23 | ~22 MB |
| Sankey 桑基图 | `public/data/sankey/sankey_season_*.json` | 23 | ~8 MB |
| KDE 距离分布 | `public/data/shot_kde_data.json` | 1 | ~15K 行 |
| 三分转型对比 | `public/data/three_point_compare/*.json` | 3 | ~8,500 行 |
| 时间-FG% 衰减 | `public/data/time_fg_base.json` | 1 | 74 行 |

## 2. 路由设计

```ts
// src/router.ts
const routes = [
  { path: '/',           component: HomePage,      meta: { label: '首页',   icon: '🏠' } },
  { path: '/dashboard',  component: DashboardPage,  meta: { label: '仪表盘', icon: '📊' } },
]
```

路由 meta 信息供 `GlobalNavBar` 使用，`PageTransition` 通过 `router.beforeEach` 拦截路由变化播放过渡动画。

### 全局导航栏 `GlobalNavBar`

- 固定定位（`position: fixed; top: 0; z-index: 100`），高度 48px
- 左侧：NBA Logo + 品牌名（点击回首页） + 首页按钮 + 仪表盘按钮
- 右侧：当前赛季标签（`📅 2019-20`）+ 当前实体标签（联盟/球队/球员，可点击清除）
- 半透明毛玻璃效果：`rgba(13,17,23,0.92)` + `backdrop-filter: blur(12px)`

### 页面过渡动画 `PageTransition`

- 路由切换时：页面黑屏（`#0d1117`）→ NBA Logo 从右飞入 → 中间停顿放大 → 继续飞出 → 黑屏消失
- 总时长 2.2s，`cubic-bezier(0.45, 0, 0.55, 1)` 缓动
- 使用 `/transition-logo.webp`（8KB）作为过渡 Logo

## 3. 首页 `HomePage.vue`

### 3.1 设计意图

品牌入口页面，提供唯一操作——进入仪表盘。

### 3.2 视觉层次

```
┌─────────────────────────────────────────────┐
│         🏀 NBA Shot Evolution                │  Hero 标题
│    1997–2020 赛季 · 470 万次投篮 · 可视分析系统  │  Hero 副标题
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │ 📊  进入仪表盘                       → │    │  入口卡片
│  │     投篮热力图 · 衰减曲线 · ...       │    │
│  └──────────────────────────────────────┘    │
│                                              │
│        点击卡片进入对应分析视图                  │  底部提示
└─────────────────────────────────────────────┘
```

### 3.3 背景层

三层堆叠（由底到顶）：

1. **背景图片** — `/bg-home.jpeg`，`cover` + `blur(2px)` + `brightness(0.35)`
2. **渐变叠加** — 双层径向渐变（蓝+橙）+ 线性暗色渐变
3. **内容层** — `z-index: 2`，卡片使用 `backdrop-filter: blur(8px)`

### 3.4 卡片交互

| 状态 | 样式 |
|------|------|
| 默认 | `rgba(255,255,255,0.06)` 背景，`0.1` 透明度边框 |
| Hover | 背景加深至 `0.12`，边框加亮，`translateY(-2px)` + 阴影 |
| 箭头 | Hover 时变蓝 `#3498db`，`translateX(4px)` 右移 |

### 3.5 组件引用

```
HomePage.vue
  无外部组件引用（纯自身渲染）
  使用 vue-router 的 useRouter().push() 导航
```

## 4. 仪表盘 `DashboardPage.vue`

### 4.1 设计意图

一屏展示全部 5 类分析图表，提供射击数据全景视图。

### 4.2 布局结构

```
┌────────────────────── 62% ───────────────┬────────── 38% ────────────┐
│                                          │                          │
│  ┌────────────────────────────────────┐  │  ┌────────────────────┐  │
│  │                                    │  │  │  KDE 距离分布演化   │  │
│  │      Hexbin 投篮热力图              │  │  │  (含双峰指数柱状图)  │  │
│  │      (左右半场独立选择器)            │  │  │  播放控制 + 统计卡片  │  │
│  │                                    │  │  └────────────────────┘  │
│  └────────────────────────────────────┘  │                          │
│  ┌────────────────────────────────────┐  │  ┌────────────────────┐  │
│  │  时间-FG% 衰减曲线                  │  │  │  三分转型分层对比    │  │
│  │  (2PT/3PT 命中率 + 出手量柱状图)     │  │  │  (先行者/过渡/落后者) │  │
│  └────────────────────────────────────┘  │  └────────────────────┘  │
│                                          │  ┌────────────────────┐  │
│                                          │  │  桑基投篮流向图      │  │
│                                          │  │  (4层 SVG 自研布局)  │  │
│                                          │  └────────────────────┘  │
└──────────────────────────────────────────┴──────────────────────────┘
```

CSS Grid 实现：
- `grid-template-columns: 62fr 38fr`
- 列内使用 `flex-direction: column` + `gap: 10px`
- 左栏：Hexbin 占 `flex: 1 1 80%`，衰减曲线占 `flex: 0 0 auto; max-height: 30%`
- 响应式断点：`max-width: 1100px` 时切换为单列

### 4.3 图表组件集成

| 区域 | 组件 | 技术 | 数据来源 |
|------|------|------|----------|
| **左栏-上** | `HexbinPage` | 自研 SVG 双层架构 | `fetchHexbinSeason()` 加载 JSON |
| **左栏-下** | `TimeFgChart` | ECharts line + bar | `fetch('/data/time_fg_base.json')` |
| **右栏-1** | `KdeChart` | ECharts line × 3 | `fetch('/data/shot_kde_data.json')` |
| **右栏-2** | `ThreePointCompareChart` | ECharts multi-line | Pinia `threePointCompare` store |
| **右栏-3** | `SankeyChart` | 自研 SVG 4层布局 | `fetchSankeySeason()` → `extractSankeyData()` |

所有数据在 `onMounted` 中并行加载（独立 try/catch，互不阻塞）。

## 5. 状态管理

### 5.1 Pinia Store: `analysisContext.ts`

单一 store，承载跨页面状态：

```
┌─ 共享层 ─────────────────────────────────────┐
│  selectedSeason        → 当前赛季             │
│  selectedTeamId/Name   → 选中的球队            │
│  selectedPlayerId/Name → 选中的球员            │
│  lastChangeSource      → 变化来源（防循环更新）  │
│  entityLabel           → computed: 实体标签     │
├─ 页面专属 ─────────────────────────────────────┤
│  selectedTimeBin       → 时间区间 (0-7)        │
│  timeLinkMode          → 'sync' | 'independent'│
│  selectedZone          → 投篮区域              │
│  selectedShotType      → '2PT' | '3PT'         │
│  isKdeAnimating        → KDE 播放状态          │
│  kdeCurrentIndex       → KDE 当前帧            │
│  playSpeed             → 播放速度              │
└───────────────────────────────────────────────┘
```

### 5.2 图表专属 Store

| Store | 文件 | 管理内容 |
|-------|------|----------|
| `useThreePointCompareStore` | `src/charts/three-point/store.ts` | 三分转型原始数据、分类、聚合、赛季列表、showMid 开关 |
| `useTimeFilterStore` | `src/charts/time-fg/store.ts` | 时间-FG% 曲线数据 |

Hexbin 和 Sankey 数据目前由各自页面组件本地管理，尚未接入 Pinia。

## 8. 组件通信模型

```
GlobalNavBar
  ├── 读取 store.selectedSeason → 显示赛季
  ├── 读取 store.entityLabel → 显示实体
  └── 写入 store.clearEntity() → 清除实体

DashboardPage
  ├── HexbinPage          ← 本地 state（未连接 store）
  ├── TimeFgChart         ← useTimeFilterStore()
  ├── KdeChart            ← fetch() + 本地 state
  │   └── 写入 store.setSeason() / store.setKdeIndex()
  ├── ThreePointCompareChart ← useThreePointCompareStore()
  └── SankeyChart         ← fetch() + props（父组件管理数据）

PageTransition
  └── router.beforeEach → 拦截导航 → 播放动画 → 释放路由
```

通信原则：
- **DashboardPage** 作为编排器，在 `onMounted` 中并行加载所有图表数据
- **Store** 用于跨图表联动（赛季选择、实体切换、时间区间高亮）
- **本地 state** 用于图表内部 loading/ready/empty 三态切换
- **Props** 用于父→子数据传递（桑基图不接受 props 之外的 store 依赖）
